## Summary

RadCN has strong Dropdown Menu primitive coverage. The package owns trigger
semantics, menu role wiring, portal movement, side/align placement, labels,
groups, separators, shortcuts, disabled items, destructive items, checkbox
items, radio groups/items, submenus, roving focus, typeahead, pointer
highlight, close-on-select, outside close, Escape/Tab behavior, collision/stage
clamping, and public class/style/data hooks. The candidate fixtures and
Playwright tests prove those primitives directly across default,
keyboard/typeahead, checkbox/radio, submenu, collision, and custom-token
scenarios.

Experiment 46 added named docs, candidate fixture, and Playwright evidence for
the four upstream Dropdown Menu example ids as user-facing compositions:
`dropdown-menu-demo`, `dropdown-menu-checkboxes`,
`dropdown-menu-radio-group`, and `dropdown-menu-dialog`.

No audit evidence requires changing the `radcn/dropdown-menu` package API.
React `useState`, Radix primitive types, `onCheckedChange`, `onValueChange`,
and `onSelect` remain upstream mechanics. In RadCN, checkbox/radio defaults
can be expressed through explicit props and later persisted by route/server
state or dependency-free browser enhancement. Dropdown-triggered Dialog opening
is app-owned composition with `radcn/dialog`, `radcn/field`, `radcn/input`,
and `radcn/textarea`, not a Dropdown Menu dependency on Dialog.

## Mapping Decisions

| Upstream concept | RadCN mapping |
| --- | --- |
| React `useState` | Map to explicit props for defaults, native form/query/server state, or small dependency-free browser enhancement for docs/fixtures that need local persistence. Do not add React. |
| `onCheckedChange` | Map checkbox defaults to `checked`; persistence is app-owned state, browser enhancement, or server-submitted state. |
| `onValueChange` | Map radio defaults to `DropdownMenuRadioGroup value`; persistence is app-owned state or server-submitted state. |
| `onSelect` | Menu item activation is package-owned close behavior. Cross-component effects, such as opening Dialog after selecting a menu item, are app-owned composition through explicit browser/server behavior. |
| `modal={false}` | RadCN Dropdown Menu is non-modal by current design and does not lock body scroll. Playwright already checks body overflow remains unlocked. |
| `Button asChild` | Map to explicit RadCN trigger components and app-authored button or icon-trigger styling. Do not emulate React Slot semantics. |
| Dialog composition | Use `radcn/dialog` for Dialog behavior. Dropdown Menu should not import or depend on Dialog. |
| Field/Input/Textarea composition | Use existing `radcn/field`, `radcn/input`, and `radcn/textarea` inside Dialog content. Dropdown Menu remains independent. |
| `lucide-react` icons | Use app-owned presentation, inline SVG, text glyphs, or the existing plain SVG icon strategy. Do not add `lucide-react`. |
| Radix primitive types | Use upstream types as read-only design reference only. Do not add Radix to RadCN. |
| Tailwind utilities | Map to RadCN classes, `class`, `style`, CSS variables, and app-authored CSS. Tailwind is not a RadCN package dependency. |
| `className` | Map to Remix UI `class`. |
| `data-slot` | Map to public `data-radcn-dropdown-menu-*`, `data-radcn-dialog-*`, `data-radcn-field-*`, `data-radcn-input`, and `data-radcn-textarea` hooks. |
| Vendor source | Use vendor files only as read-only evidence. RadCN package code, docs, fixtures, tests, and pnpm dependency resolution must not depend on `vendor/`. |
