## Summary

RadCN has strong Input OTP primitive coverage. `radcn/input-otp` owns
`maxLength`, slot groups, separators, slot character mirroring, active slot and
caret state, digit-only and alphanumeric pattern filtering, paste filtering,
keyboard movement, disabled state, invalid ARIA state, native input value,
native form submission/reset, controlled/default value rendering, custom
classes/styles, public data hooks, and dependency-free browser enhancement.
The candidate fixtures and Playwright tests prove those primitive mechanics
directly across default, separator, digits-only, alphanumeric, four-digit,
disabled, invalid, form-submit/reset, paste, and custom-token scenarios.

Experiment 48 added named docs, candidate fixture, and Playwright evidence for
the four upstream Input OTP example ids as user-facing compositions:
`input-otp-demo`, `input-otp-pattern`, `input-otp-separator`, and
`input-otp-controlled`.

No audit evidence requires changing the `radcn/input-otp` package API. React
`useState`, upstream `input-otp` package imports, `OTPInput` context,
`lucide-react` separator icons, Tailwind utilities, and vendor source remain
upstream mechanics or app-owned presentation. Controlled entered-value display
is app-owned state or dependency-free enhancement around the native input
value/change event.

`input-otp-form` and `otp-*` JSON payloads exist upstream as adjacent form and
block recipes. They depend on Form, Button, Card, Field, toast, and layout
composition, so they should remain outside this four-example cluster unless a
future inventory explicitly reclassifies them.

## Mapping Decisions

| Upstream concept | RadCN mapping |
| --- | --- |
| React `useState` | Map to native input value, server/route state, or app-owned dependency-free browser enhancement for visible display text. Do not add React. |
| `value` | Keep as explicit RadCN prop for rendered/controlled initial value and native input value. |
| `onChange` | Map package-owned value updates to native `input` events and `radcn-input-otp-change`; visible feedback remains app-owned. |
| Upstream `input-otp` package and `OTPInput` context | RadCN reimplements the required behavior with a native input plus slots. Do not add upstream `input-otp` as a dependency. |
| `REGEXP_ONLY_DIGITS_AND_CHARS` | Use RadCN's exported constant and native pattern filtering. |
| `lucide-react` separator icon | Separator icon/presentation is app-owned; RadCN's separator can render text or children without a lucide dependency. |
| Tailwind utilities | Map to RadCN classes, `class`, `containerClass`, `style`, CSS variables, and app-authored CSS. Tailwind is not a RadCN package dependency. |
| `className` and `containerClassName` | Map to Remix UI `class` and RadCN `containerClass`. |
| `data-slot` | Map to public `data-radcn-input-otp-*` hooks. |
| Vendor source | Use vendor files only as read-only evidence. RadCN package code, docs, fixtures, tests, and pnpm dependency resolution must not depend on `vendor/`. |
| `input-otp-form` and `otp-*` blocks | Treat as adjacent form/block evidence. Their Form, Button, Card, toast, and product layout behavior belong to future form/block example work, not this four-example Input OTP cluster. |
